# Contributing to SkriptArchive Docs

## 1. Fork & Branch
- Fork the repository.
- Create a branch with a descriptive name, e.g., `add-fly-command-tutorial`.

## 2. Markdown Format
- Use the template for script pages:
  - Title
  - Description
  - Version
  - Author
  - Tags
  - Code block: ```skript ... ```
- Link to scripts repo: `[Fly Command](https://github.com/SkriptArchive/scripts/blob/main/fly-command.sk)`

## 3. Pull Request
- PR target: `main` branch  
- PR title format: `[Snippet] Fly Command`  
- PR body: explain what was added or updated.

## 4. Review & Merge
- Maintainers check for correct Markdown format, functional links, proper code blocks.
